# llm-quality-harness

**Инструмент, который отвечает на вопрос: «А ваш ИИ-агент точно не врёт?»**

Эвал-харнесс для LLM-агентов и RAG-систем: набор кейсов, автоматический прогон, метрики, детекция галлюцинаций, регрессия промптов. Работает с любой моделью через единый слой провайдеров.

---

## Зачем это нужно

Агента на LLM сегодня собирает кто угодно за выходные. Проблема начинается на второй неделе, когда бизнес задаёт три вопроса, и на них нечем ответить:

1. **«Он вообще правильно отвечает?»** Без замера это вопрос веры. Демо на пяти удачных запросах ничего не доказывает.
2. **«Мы поменяли промпт - стало лучше или хуже?»** Без регрессии это гадание. Правка, которая чинит один сценарий, тихо ломает три других.
3. **«Вышла новая версия модели, можно переезжать?»** Без прогона это лотерея с продом в качестве ставки.

Эти три вопроса и есть причина, по которой пилоты с ИИ не доходят до промышленной эксплуатации. Не потому что модель слабая, а потому что **никто не может доказать, что она работает**.

Харнесс превращает «мне кажется, стало лучше» в таблицу с цифрами.

## Что он меряет

| Метрика | Что показывает | Почему бизнесу важно |
|---|---|---|
| `pass@k` | Доля кейсов, где агент дал верный ответ | Базовое качество: можно ли вообще выпускать |
| `Hit@k` | Нашёл ли RAG нужный документ в топ-k | Отделяет ошибку поиска от ошибки генерации |
| `MRR` | Насколько высоко в выдаче нужный документ | Качество ранжирования базы знаний |
| `groundedness` | Опирается ли ответ на найденные источники | **Прямая мера галлюцинаций** |
| `citation_accuracy` | Ссылается ли агент на реальный источник | Можно ли проверить ответ, а не верить на слово |
| `refusal_rate` | Как часто агент честно говорит «не знаю» | Молчание лучше уверенного вранья |
| `latency_p95`, `cost_per_case` | Скорость и деньги | Экономика внедрения |

Ключевая идея: **`pass@k` без `groundedness` бесполезен**. Агент может дать верный ответ, выдумав его - и на следующем вопросе выдумает неверный. Мы меряем не только *что* ответил, но и *откуда взял*.

## Как устроено

```
cases/*.yaml        - кейсы: вопрос, эталон, ожидаемый источник, теги
src/providers.py    - model-agnostic слой (OpenAI, Anthropic, локальные, GigaChat/YandexGPT)
src/retrieval.py    - подключение вашего RAG (интерфейс на 1 метод)
src/judge.py        - LLM-судья: сверяет ответ с эталоном по рубрике
src/groundedness.py - разбирает ответ на утверждения и проверяет каждое по источникам
src/metrics.py      - расчёт метрик
src/harness.py      - прогон, кэш, параллелизм
src/report.py       - отчёт в Markdown/HTML + сравнение с baseline
```

Агент подключается через два интерфейса - `answer(question) -> Answer` и `retrieve(query) -> [Doc]`. Больше харнесс о вашей системе ничего знать не хочет.

## Быстрый старт

```bash
pip install -r requirements.txt
export LLM_API_KEY=...

# прогон на демо-кейсах
python -m src.harness --cases cases/support_agent.yaml --report out/report.md

# сравнить с предыдущим прогоном (регрессия промпта)
python -m src.harness --cases cases/support_agent.yaml --baseline out/baseline.json
```

Отчёт: [examples/report_example.md](examples/report_example.md)

## Регрессия промптов

Промпт - это код. К нему применимы те же правила: версионирование, тесты, ревью, откат.

```bash
# зафиксировали baseline на текущем промпте
python -m src.harness --cases cases/support_agent.yaml --save-baseline out/baseline.json

# правим промпт, гоняем, видим дельту по каждой метрике и по каждому кейсу
python -m src.harness --cases cases/support_agent.yaml --baseline out/baseline.json
```

Отчёт покажет не только сводные цифры, но и **какие конкретно кейсы сломались** - это и есть то, чего не видно на глаз.

## Смена модели

```bash
python -m src.harness --cases cases/support_agent.yaml --model gpt-4o --save-baseline out/gpt4o.json
python -m src.harness --cases cases/support_agent.yaml --model claude-sonnet --baseline out/gpt4o.json
```

Одни и те же кейсы, разные модели, одна таблица. Решение о переезде принимается по цифрам, а не по маркетингу вендора.

## Кейсы

Кейс - это не «вопрос-ответ», а проверяемое утверждение:

```yaml
- id: refund-window
  question: Сколько дней даётся на возврат товара?
  expected: 14 дней с момента получения
  expected_source: policy/returns.md#sroki
  tags: [политика, критичный]
  must_cite: true       # ответ без ссылки на источник = провал
  
- id: unknown-policy
  question: Какая у вас политика по возврату из-за рубежа?
  expected_behavior: refuse   # в базе этого нет - агент обязан признать незнание
  tags: [негативный]
```

**Негативные кейсы обязательны.** Набор, где агент должен только отвечать, ничего не проверяет. Половина ценности - в проверке, что он молчит там, где не знает.
